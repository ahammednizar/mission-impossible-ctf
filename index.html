// setup_lab.sh

Setup Instructions

Get your lab environment ready in minutes.

Step 1 — Prerequisites ▼

Before you begin, ensure you have:

  • VirtualBox 6.0+ — Download Here
  • Kali Linux or any pentesting OS — Download Here
  • At least 4GB RAM and 20GB free disk space
  • Basic Linux command line familiarity
  • Familiarity with nmap, netcat, etc.
Step 2 — Import OVA File ▼
  1. Download Project_Mission_Impossible.ova
  2. Open VirtualBox → File → Import Appliance
  3. Select the downloaded .ova file
  4. Review settings (2GB RAM, 2 CPU recommended)
  5. Click Import — takes 5–10 minutes
Step 3 — Network Configuration ▼

Option A: Bridged Adapter (Recommended)

  1. Right-click VM → Settings → Network
  2. Change "Attached to" → Bridged Adapter
  3. VM will get an IP from your router (e.g. 192.168.1.x)

Option B: Host-Only Adapter (Isolated)

  1. File → Host Network Manager → Create network
  2. VM Settings → Network → Host-Only Adapter
  3. VM will be isolated from your home network
Step 4 — Start the Machine ▼
  1. Select Project Mission Impossible VM
  2. Click Start — wait ~30 seconds to boot
  3. Find IP by running ip addr show on the VM console
bash
# Or scan your network from Kali
nmap -sn 192.168.1.0/24
Step 5 — Begin Your Mission ▼
bash
# Scan the target
nmap -sV -sC -p- <TARGET_IP>

# Visit the web interface
firefox http://<TARGET_IP>

Your mission begins at port 80. Follow the clues and good luck, Agent! 🎯

// vuln_analysis.sh

Vulnerabilities Explained

CRITICAL Unrestricted File Upload ▼

Location: /upload.php on port 8080

The file upload accepts any file type without validation, allowing PHP scripts to be uploaded.

Vulnerable Code:

php
<?php
// NO FILE TYPE VALIDATION!
if (move_uploaded_file($_FILES["fileToUpload"]["tmp_name"], $target_file)) {
    echo "File uploaded successfully!";
}
?>

Fix:

php
$allowed = ['jpg','jpeg','png','gif','pdf'];
$ext = strtolower(pathinfo($target_file, PATHINFO_EXTENSION));
if (!in_array($ext, $allowed)) { die("Invalid file type!"); }
// Also validate MIME type with finfo_file()
HIGH Information Disclosure via HTML Comments ▼

Credentials stored in HTML comments visible to anyone via View Source.

Vulnerable Code:

html
<!-- Username: luther_stickell  Password: HackerInChief_1996 -->

Fix:

  • Never store credentials in client-side code
  • Remove all sensitive comments before deployment
  • Use environment variables / secrets manager
CRITICAL SUID Binary Misconfiguration ▼

Root-owned binary with SUID bit calls system(), enabling PATH hijack for privesc.

Vulnerable Code:

c
setuid(0); setgid(0);
system("/bin/cat /root/FINAL_MISSION.txt"); // PATH hijackable

Fix:

  • Avoid SUID binaries — use sudo with explicit rules instead
  • Never use system() in SUID programs
  • Use execve() with full absolute paths
  • Audit regularly: find / -perm -4000 2>/dev/null
MEDIUM Weak Password Policy ▼

Predictable, thematic passwords across accounts:

  • ethan_hunt : IMF_Protocol_2024
  • luther_stickell : HackerInChief_1996

Fix:

  • Enforce 12+ character complex passwords
  • Enable MFA on all accounts
  • Prefer SSH key-based authentication

// faq.sh

Frequently Asked Questions

Can't find the VM's IP address? ▼
  1. Login to VM console → run ip addr show
  2. From Kali → nmap -sn 192.168.1.0/24
  3. Use arp-scan -l
  4. Check router's DHCP client list
Reverse shell not connecting? ▼
  1. Verify your Kali IP in the PHP shell file
  2. Check firewall: sudo ufw status
  3. Ensure listener is running: nc -lvnp 4444
  4. Access the shell URL directly in browser
  5. Check Apache error logs for PHP errors
Can't find the SUID binary? ▼
bash
# Try these locations:
ls -la /tmp/secret_mission
ls -la /usr/local/bin/secret_mission
ls -la /home/ilsa_faust/secret_mission

# Search for it:
find / -name "secret_mission" 2>/dev/null
find / -perm -4000 2>/dev/null | grep secret
What if I get stuck? ▼
  • Read all mission briefing files — they contain clues
  • Always View Page Source on web pages
  • Check .bash_history files
  • Look for hidden files with ls -la
  • Refer to the complete walkthrough above
Is this machine safe to run? ▼

Only safe in isolated environments:

  • ✅ VirtualBox with Host-Only or NAT network
  • ✅ Isolated home lab without internet
  • ❌ Do NOT expose to the internet
  • ❌ Do NOT run on public cloud without isolation
